## 1. 绪论

### 1.1 电商导购场景下的智能问答需求
- 传统电商搜索的局限性（关键词匹配、无法理解图片）
- 用户"拍照识图"的真实场景需求
- 大模型时代下的智能导购新范式

### 1.2 大模型 + RAG + Agent 的技术趋势
- 大语言模型的兴起与局限（幻觉、知识截止）
- RAG（检索增强生成）的提出与演进
- Agent 编排框架的发展（LangChain → LangGraph）

### 1.3 论文主要工作与创新点
- 基于 LangGraph StateGraph 的多 Agent 编排设计
- 四层记忆系统（工作/情景/语义/感知）
- 多模态 RAG 检索（CLIP + BGE-M3 + Qdrant + RRF）
- RAGAS 质量评估流水线
- GSAP 前端动效优化

## 3. 系统需求分析

### 3.1 业务需求
- 拍照识图：用户拍摄商品图片，系统识别并推荐同类商品
- 导购推荐：基于候选商品生成推荐理由和知识引用
- 澄清交互：低置信度时主动追问用户偏好

### 3.2 功能需求
- 图片上传与管理
- 多模态商品检索
- 流式回答生成（SSE）
- 候选商品展示
- 引用展示
- 澄清交互

### 3.3 非功能需求
- 实时性：全链路响应 P95 < 5s
- 可评测性：支持 RAGAS 自动化评估
- 可演示性：Docker 一键启动
- 可扩展性：单一品类可扩展到多品类

## 5. 系统详细设计与实现

### 5.1 意图识别模块实现
- LLM 驱动的意图分类
- 意图类型定义（find_similar / ask_product / compare / unclear）
- 识别准确性优化

### 5.2 Plan & Solve 计划生成实现

- 动态计划调整策略

### 5.3 基于 ReAct 的 Agent 循环实现
- Think（LLM 推理当前状态）
- Act（调用 RAG 工具/生成回答）

### 5.4 Reflection 自反思修正实现

- 5.5.4 Qdrant 向量检索与 RRF 混合排序

### 5.6 四层记忆系统实现

- 5.6.3 语义记忆（Qdrant 向量库 + 偏好提取）

### 5.7 流式生成与澄清交互
