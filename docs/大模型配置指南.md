# 大模型配置统一指南

## 🎯 核心理念

**修改一个文件，全局生效**

所有大模型配置都集中在 `backend/.env` 文件中，修改后整个项目（后端 API、评估脚本、Agent 等）都会自动使用新配置。

---

## 📁 配置文件位置

```
Agent/
└── backend/
    └── .env                    # 唯一需要修改的文件
```

---

## ⚙️ 配置项说明

### 主模型配置

```bash
# 主模型 API 密钥
LLM_API_KEY=sk-your-api-key-here

# 主模型 API 地址
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# 主模型名称
LLM_MODEL=qwen-plus

# 其他参数（可选）
LLM_MAX_TOKENS=2048
LLM_TEMPERATURE=0.7
LLM_TIMEOUT=60
```

### Fallback 模型配置（可选）

当主模型失败时自动切换到备用模型：

```bash
# 备用模型 API 密钥（空则复用主模型）
LLM_FALLBACK_API_KEY=

# 备用模型 API 地址（空则复用主模型）
LLM_FALLBACK_BASE_URL=

# 备用模型名称（空则不启用 fallback）
LLM_FALLBACK_MODEL=
```

## 🏗️ 配置读取流程

```
backend/.env
    ↓
backend/app/config.py (Settings 类)
    ↓
    ├─→ backend/app/graph/llm.py (ResilientLLM)
    │       ↓
    │       └─→ Agent 编排、对话生成
    │
    ├─→ backend/app/agent/react_loop.py
    │       ↓
    │       └─→ ReAct 循环
    │
    └─→ rag/eval/eval_config.py (EvalConfig 类)
            ↓
            └─→ RAGAS 评估脚本
```

**所有组件都从 `backend/.env` 读取配置，修改一处即可全局生效。**

## ⚠️ 注意事项

1. **API 密钥安全**
   - `.env` 文件已在 `.gitignore` 中，不会提交到 Git
   - 不要将 `.env` 文件分享给他人

2. **模型兼容性**
   - 确保模型支持 OpenAI 兼容的 API 格式
   - 不同模型的参数格式可能略有差异

3. **Fallback 配置**
   - Fallback 模型的 API 密钥和地址可以与主模型不同
   - 建议配置一个可靠的备用模型

4. **评估脚本**
   - RAGAS 评估脚本也使用相同的配置
   - 修改 `.env` 后，评估脚本会自动使用新模型
