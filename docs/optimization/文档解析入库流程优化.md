# 文档解析与入库流程优化方案

> 分析日期：2026-07-06
> 目标：将现有简易 DocLoader 改造为企业级文档解析与入库流程

---

## 一、现状分析

### 1.1 当前实现（简易版）

```
docs/ → DocLoader → embed_text → Qdrant.upsert(documents)
```

- ✅ **优点**：单类单函数，10-30 份文档 PoC 够用
- ❌ **缺点**：
  1. **PDF 解析弱**：只支持 pypdf 提取纯文本，扫描件 OCR 不支持
  2. **分块无语义**：固定 512 字符切分，跨段落切断
  3. **无幂等性**：重导同一文档会重复入库（顺序 int ID 冲突）
  4. **无图片处理**：MD 中的本地图片 URL 不替换
  5. **无结构感知**：标题、表格、列表无差异化处理
  6. **embedding 慢**：per-chunk 逐个调用，无批量

### 1.2 PythonProject16 实现（企业级）

```
docs/ → entry → pdf_to_md (MinerU/PyPDF2) → md_img (OSS+LLM) →
         document_split (3阶段) → item_name_recognition (LLM) →
         bge_embedding (batch) → qdrant (幂等删除)
```

- ✅ **优点**：专业级 PDF 处理（OCR+表格+版式）、结构感知分块、幂等入库
- ❌ **缺点**：需要 MinerU API Key 或 PyPDF2、阿里云 OSS、LLM 调用次数多

## 二、关键差距与影响

| 差距 | 当前实现 | 企业级需求 | 影响程度 |
|------|---------|-----------|---------|
| **PDF 扫描件** | 提取失败 | 需要 OCR | 🔴 高 |
| **表格识别** | 文本混乱 | 需保留表格结构 | 🔴 高 |
| **分块语义** | 跨段切断 | 按章节切 | 🔴 高 |
| **幂等性** | 重导冲突 | 按 doc_id 删除旧数据 | 🔴 高 |
| **图片处理** | 不处理 | 上传 OSS + 摘要 | 🟡 中 |
| **批量 embedding** | 逐条调 | 批量 50 条/批 | 🟡 中 |
| **商品名识别** | 不需要 | Li-Ning 单品牌可省 | 🟢 低 |

## 三、优化方案设计

### 3.1 总体架构

**5 阶段流水线**（基于 PythonProject16 简化）：

```
docs/
  ↓ DocLoader（格式检测 + 解析）
stage_1: parse
  - PDF: MinerU 云 / pypdf 降级
  - MD: 直接读
  - DOCX: python-docx
  - XLSX: openpyxl
  - CSV: csv DictReader
  ↓
stage_2: clean
  - 去空 < 10 字
  - 去重 hash
  - 表格/标题/正文类型标注
  ↓
stage_3: chunk (3 阶段)
  - 阶段 1: 按 MD 标题切分 (#/##/###)
  - 阶段 2: 超长章节按段落 \n\n 二次切
  - 阶段 3: 超长段落按句子。！？三次切
  - 阶段 4: 短块 (<200 字) 与相邻块合并
  ↓
stage_4: embed
  - 批量调用 BGE-M3（50 条/批）
  ↓
stage_5: upsert (幂等)
  - 按 doc_id 删除旧数据
  - upsert 新数据（UUID 主键）
```

### 3.2 阶段 1：PDF 解析（双路径）

```python
def parse_pdf(file_path: str) -> str:
    # 路径 A：MinerU 云解析（高质量，支持扫描+表格+OCR）
    if mineru_api_key:
        return mineru_parse(file_path)
    # 路径 B：pypdf 本地降级（仅数字版 PDF 文本提取）
    return pypdf_parse(file_path)
```

**决策点**：
- 你的 10-30 份文档是数字版还是扫描件？
  - 数字版（可复制文本）→ 路径 B 够用，省 API 费用
  - 扫描件 → 必须路径 A，需要申请 MinerU API Key
- 是否含表格密集内容？
  - 是 → 必须路径 A（pypdf 表格提取乱）
  - 否 → 路径 B 即可

### 3.3 阶段 3：结构感知分块（核心改造）

```python
def chunk_by_structure(md_text: str, max_len: int = 512) -> list[DocChunk]:
    """3 阶段分块"""
    # 阶段 1：按标题切分
    sections = split_by_heading(md_text)
    # 阶段 2：超长章节二次切分
    chunks = []
    for title, content in sections:
        if len(content) <= max_len:
            chunks.append(make_chunk(title, content))
        else:
            # 按段落 \n\n 切
            for para in content.split("\n\n"):
                if len(para) <= max_len:
                    chunks.append(make_chunk(title, para))
                else:
                    # 阶段 3：按句子切
                    for sentence in split_by_sentence(para):
                        chunks.append(make_chunk(title, sentence))
    # 阶段 4：短块合并
    chunks = merge_short_chunks(chunks, min_len=200)
    return chunks
```

**效果**：相比固定 512 字符切分，RAGAS Context Precision 预计 +10-20%

### 3.4 阶段 5：幂等性入库

```python
def ingest_with_idempotent(doc_id: str, chunks: list[DocChunk]):
    """按 doc_id 幂等入库"""
    client = get_qdrant_client()
    # 删除同 doc_id 的旧数据
    client.delete(
        collection_name="documents",
        points_selector=Filter(must=[
            FieldCondition(key="doc_id", match=MatchValue(value=doc_id))
        ])
    )
    # upsert 新数据
    points = [
        PointStruct(id=str(uuid.uuid4()), vector={...}, payload={...})
        for chunk in chunks
    ]
    client.upsert(collection_name="documents", points=points)
```

**效果**：重导同名文档不会产生重复数据

## 四、实施计划

### Phase 1：基础改造（Day 1，~150 行）
1. **DocLoader 扩展**：
   - `_parse_pdf_with_mineru()` 路径 A
   - `_parse_pdf_with_pypdf()` 路径 B
   - `_chunk_by_heading()` 3 阶段分块
2. **入库脚本改造**：
   - 改 `ingest_documents.py` 加幂等删除
   - 用 UUID 主键
3. **验证**：用 3-5 份混合格式测试文档入库，统计 chunk 数

### Phase 2：质量优化（Day 2，~100 行）
1. **批量 embedding**：50 条/批调用 BGE-M3
2. **短块合并**：相邻同 parent_title 的 < 200 字块合并
3. **元数据增强**：chunk payload 加 `parent_title`、`chunk_index`、`token_count`

### Phase 3：可观测性（Day 3，~50 行）
1. **入库日志**：每个文件入库耗时、chunk 数、失败原因
2. **错误隔离**：单个文件失败不阻塞整体入库
3. **本地备份**：入库成功后写 `ingest_backup.jsonl`

## 五、与现有 PoC 的兼容性

**保持不变**：
- `DocLoader` 类接口（`load()` / `load_directory()`）
- `DocumentRetriever` 接口
- `documents` Qdrant 集合名
- `docs_search_node` / `docs_retrieve_node`

**增强**：
- `_chunk()` → `_chunk_by_heading()`（内部方法，不影响外部）
- 入库脚本加 idempotent

## 六、风险与缓解

| 风险 | 缓解 |
|------|------|
| MinerU API 不稳定 | 路径 B 降级（pypdf）+ 配置开关 |
| 标题感知分块切错 | 单元测试覆盖 10 个典型 MD |
| 批量 embedding 失败 | 单条 fallback + 重试 |
| UUID 索引膨胀 | 按 doc_id 删除旧数据（幂等） |
