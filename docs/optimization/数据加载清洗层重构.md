## 二、开源方案对比

### Unstructured.io（文档解析的事实标准）

GitHub 源码验证：`unstructured/partition/` 下 **27 个格式模块**：

```
csv.py      doc.py      docx.py     email.py    epub.py
image.py    json.py     md.py       msg.py      ndjson.py
odt.py      org.py      pdf.py      ppt.py      pptx.py
rst.py      rtf.py      text.py     tsv.py      xlsx.py
xml.py      audio.py    auto.py     api.py
```

**核心架构**：

```
用户文件（任意格式）
    │
    ▼
auto.py → 检测扩展名 → 路由到对应 partition_xxx()
    │
    ├── partition_pdf()
    │   ├── strategy="fast"    → pymupdf 提取文字+表格
    │   ├── strategy="hi_res"  → 每页转PNG → OCR → 布局重建
    │   └── strategy="auto"    → 先 fast，文字少回退 hi_res
    │
    ├── partition_docx()       → python-docx 解析XML
    ├── partition_xlsx()       → openpyxl / pandas
    ├── partition_md()         → Python markdown 库解析 AST
    ├── partition_image()      → pytesseract / paddleocr
    ├── partition_audio()      → whisper 语音转文字
    └── ...

    │
    ▼
统一输出: List[Element]
    ├── Title          (标题)
    ├── NarrativeText  (正文段落)
    ├── Table          (表格)
    ├── ListItem       (列表项)
    ├── Image          (图片+描述)
    └── CompositeElement (混合)
    │
    ▼
Chunking → Embedding → Vector DB
```

**一句话**：`from unstructured.partition.auto import partition; elements = partition("任意文件.pdf")` 全部搞定。

### 你需要写的只是包装层

```python
"""
doc_loader.py — 基于 Unstructured 的统一文档加载器

依赖: pip install unstructured
完整安装: pip install "unstructured[pdf,docx,xlsx,pptx,md]"
"""

from pathlib import Path
from typing import List, Optional
from dataclasses import dataclass, field


@dataclass
class DocChunk:
    """入库就绪的文档片段"""
    content: str                    # 文本内容
    chunk_type: str                 # "title"|"paragraph"|"table"|"list_item"
    source_file: str                # 来源文件路径
    page_number: Optional[int] = None
    metadata: dict = field(default_factory=dict)


class DocLoader:
    """统一文档加载器：自动检测格式 → 解析 → 清洗 → 分块"""

    def __init__(self, chunk_size: int = 512, chunk_overlap: int = 50):
        self.chunk_size = chunk_size
        self.chunk_overlap = chunk_overlap

    def load(self, file_path: Path) -> List[DocChunk]:
        """加载单个文件（自动检测格式）"""
        from unstructured.partition.auto import partition

        # Step 1: Unstructured 自动解析
        elements = partition(
            filename=str(file_path),
            strategy="auto",              # PDF: fast/hi_res/auto
            include_page_breaks=True,     # 保留分页信息
        )

        # Step 2: 清洗
        elements = self._clean(elements)

        # Step 3: 转为统一结构
        chunks = []
        for el in elements:
            text = str(el).strip()
            if not text:
                continue

            chunks.append(DocChunk(
                content=text,
                chunk_type=self._map_type(type(el).__name__),
                source_file=str(file_path),
                page_number=getattr(el, 'metadata', {}).get('page_number'),
                metadata={
                    "source": str(file_path),
                    "element_type": type(el).__name__,
                },
            ))

        # Step 4: 分块（长段落切分，短段落合并）
        chunks = self._chunk(chunks)

        return chunks

    def _clean(self, elements: list) -> list:
        """清洗管道"""
        cleaned = []
        seen = set()
        for el in elements:
            text = str(el).strip()
            # 过滤空内容
            if not text:
                continue
            # 过滤过短无意义片段
            if len(text) < 5:
                continue
            # 去重（前 100 字符 hash）
            h = hash(text[:100])
            if h in seen:
                continue
            seen.add(h)
            cleaned.append(el)
        return cleaned

    def _map_type(self, element_type: str) -> str:
        """Unstructured Element → 你的类型映射"""
        mapping = {
            "Title": "title",
            "Header": "title",
            "NarrativeText": "paragraph",
            "Text": "paragraph",
            "Table": "table",
            "ListItem": "list_item",
            "CompositeElement": "paragraph",
        }
        return mapping.get(element_type, "paragraph")

    def _chunk(self, chunks: List[DocChunk]) -> List[DocChunk]:
        """分块：过长段落切分"""
        result = []
        for c in chunks:
            if len(c.content) <= self.chunk_size:
                result.append(c)
            else:
                # 简单滑动窗口分块
                for i in range(0, len(c.content), self.chunk_size - self.chunk_overlap):
                    sub = c.content[i:i + self.chunk_size]
                    if sub.strip():
                        result.append(DocChunk(
                            content=sub.strip(),
                            chunk_type=c.chunk_type,
                            source_file=c.source_file,
                            page_number=c.page_number,
                            metadata=c.metadata,
                        ))
        return result

    def load_directory(self, dir_path: Path) -> List[DocChunk]:
        """批量加载目录下所有支持的文件"""
        all_chunks = []
        for fp in dir_path.rglob("*"):
            if fp.is_file() and fp.suffix.lower() in {
                ".pdf", ".docx", ".doc", ".xlsx", ".xls", ".csv",
                ".pptx", ".ppt", ".html", ".htm", ".md", ".txt",
                ".rst", ".epub", ".odt", ".rtf", ".json",
                ".png", ".jpg", ".jpeg", ".tiff", ".bmp",
            }:
                try:
                    chunks = self.load(fp)
                    all_chunks.extend(chunks)
                    print(f"[OK] {fp.name}: {len(chunks)} 片段")
                except Exception as e:
                    print(f"[跳过] {fp.name}: {e}")
        return all_chunks
```

### 与现有管线的集成

```python
# ingest_data_enhanced.py 的替代版

from rag.doc_loader import DocLoader
from rag.embedding import embed_text
from rag.db_client import get_qdrant_client

def ingest_from_directory(data_dir: str = "rag/data"):
    loader = DocLoader(chunk_size=512, chunk_overlap=50)
    chunks = loader.load_directory(Path(data_dir))

    client = get_qdrant_client()

    for i, chunk in enumerate(chunks):
        embedding = embed_text(chunk.content)
        client.upsert(
            collection_name="products",
            points=[{
                "id": f"doc_{i}",
                "vector": {"text": embedding},
                "payload": {
                    "content": chunk.content,
                    "type": chunk.chunk_type,
                    "source_file": chunk.source_file,
                    "page": chunk.page_number,
                    **chunk.metadata,
                }
            }]
        )

    print(f"入库完成: {len(chunks)} 片段")
```

## 四、各格式的底层解析原理（面试重点）

### PDF — 最复杂的格式

| 策略 | 引擎 | 适用 | 原理 |
|------|------|------|------|
| `fast` | pymupdf | 原生 PDF（非扫描件） | 解析 content stream → 提取文字坐标 → 按 y/x 排序重建阅读顺序 |
| `hi_res` | pymupdf + pytesseract | 扫描件/图片型 PDF | 每页转 300dpi PNG → OCR 识别 → layoutparser 检测区块 → 合并文字层 |
| `auto` | 自适应 | 不确定类型时 | 先 fast 提取，文字量 < 阈值 → 回退 hi_res |

**面试考点**：
> "PDF 不是纯文本——是二进制 page tree。解析的核心挑战是阅读顺序重建（双栏、表格混排）和扫描件 OCR。我们用 pymupdf 做基础提取，扫描件用 hi_res 策略回退到 Tesseract/PaddleOCR。"

### Markdown

| 引擎 | 原理 |
|------|------|
| Python `markdown` 库 | 解析为 AST → 遍历节点 → Title/NarrativeText/Code/ListItem/Table 分类输出 |

**面试考点**：
> "Markdown 解析比 PDF 简单，因为有明确的语法标记（#标题、```代码块、|表格|）。我们通过 AST 遍历保留文档结构层级，表格、代码块单独成块，避免被切分破坏语义。"

### Word (DOCX)

| 引擎 | 原理 |
|------|------|
| `python-docx` | 解析 XML（docx 本质是 ZIP 压缩的 XML）→ 按 paragraph/table/style 提取，保留标题层级 |

### Excel (XLSX)

| 引擎 | 原理 |
|------|------|
| `openpyxl` / `pandas` | 读取 sheet → 按行遍历单元格 → 有策略地拼接成文本（含列名） |

### 图片 (PNG/JPG)

| 策略 | 引擎 | 原理 |
|------|------|------|
| OCR | pytesseract / PaddleOCR | 文字检测 → 文字识别 → 输出文本 |
| 视觉理解 | 多模态 LLM | 不仅 OCR，还理解图片内容（"这是一张羽毛球鞋底部的碳板特写"） |

## 五、面试问答

> **Q: 你们的数据源是什么格式？怎么处理的？**
>
> 我们的数据加载层基于 Unstructured.io 构建——这是一个处理非结构化文档的 Python 库，支持 20+ 格式。我们封装了 `DocLoader` 统一入口，`load()` 方法自动检测文件格式、路由到对应 Parser、清洗、分块。核心流程：Unstructured 解析 → 去空/去重/去噪 → 语义分块 → 元数据附加 → 向量化入库。

> **Q: PDF 解析的难点在哪？怎么解决的？**
>
> PDF 本质是二进制 page tree，不是文本流。三个难点：
> 1. **阅读顺序**：PDF content stream 不保证从上到下从左到右，pymupdf 通过文字坐标按 y→x 排序重建；
> 2. **表格**：表格在 PDF 里是线条+散落文字，需要线检测（pymupdf find_tables）或空白间距推断（camelot）；
> 3. **扫描件**：图片型 PDF 需要 OCR，用 hi_res 策略：转 300dpi PNG → Tesseract/PaddleOCR 识别 → layoutparser 区块检测。
>
> Unstructured 把这三种情况封装成了 fast/hi_res/auto 三种策略，auto 模式下先尝试文字提取，不足时自动回退 OCR。

> **Q: 为什么用 Unstructured 而不是自己写？**
>
> 文档解析是一个"看起来简单、做起来全是坑"的领域。每种格式都有边界情况——PDF 的扫描件、Word 的嵌套表格、Excel 的合并单元格、Markdown 的 HTML 内嵌。Unstructured 是 GitHub 16k+ stars 的开源项目，这些边界情况已经被社区验证过。我们的价值在业务层（商品数据清洗、结构化属性提取），不在文档解析层。

> **Q: Markdown 和 PDF 的解析有什么区别？**
>
> Markdown 有明确的语法标记（`#` 标题、`|` 表格、` ``` ` 代码块），通过 AST 解析可以精确还原文档结构，不会丢失语义层级。PDF 没有结构标记，需要从二进制 content stream 中提取文字坐标来"猜测"阅读顺序，表格和图片更是需要额外的检测算法。Markdown 解析是确定性的，PDF 解析是概率性的。
